Meta Information

  1. Page Information
    • application-name

      • <meta name="application-name" content="Your App Name">

    • author

      • <meta name="author" content="Your Name">

    • description

      • <meta name="description" content="Page Description">

      The description meta tag can be used by various search engines while indexing your web page for searching purpose. Usually, the description contained within the meta tag is the short summary that shows up under the page/website's main title in the search engine results. Google usually uses only the first 20-25 words of your description.

    • generator

      • <meta name="generator" content="HTML Generator 1.42">

      Identifies one of the software packages used to generate the document. Only to be used for pages where the markup is automatically generated.

    • keywords

      • <meta name="keywords" content="Keyword1, Keyword2">

      The keywords meta tag is sometimes used by search engines to know the search query which is relevant to your web page. As a rule of thumb, it is probably a good idea to not add too many words, as most search engines that use this meta tag for indexing will only index the first ~20 words. Make sure that you put the most important keywords first.

    • robots

      • <meta name="robots" content="noindex">

      The robots meta tag tells search engines what they are allowed (or not allowed) to do with a web page. You can control whether search engines should index the page or follow the links on it. Below are common values:

      • all - Default; equivalent to index, follow
      • noindex - Prevent indexing of the page
      • nofollow - Prevent search engines from following any links on the page
      • none - Equivalent to noindex, nofollow
      • noarchive - Don't allow cached versions in search results
      • nosnippet - Don't show a snippet in search results
      • notranslate - Don't allow translation options in search
      • noimageindex - Prevent indexing of images on the page
      • unavailable_after - Remove the page from search after a specific RFC 850 date/time

      Note: Explicitly specifying index and follow is not required, as these are assumed by default if not overridden.

  2. Character Encoding

    3. The charset attribute specifies the character encoding for the HTML document and needs to be a valid character encoding (examples include windows-1252, ISO-8859-2, Shift_JIS, and UTF-8). UTF-8 (Unicode) is the most widely used and should be used for any new project.

    <meta charset="UTF-8">
<meta charset="ISO-8859-1">

    All browsers have always recognized the <meta charset> form, but if you for some reason need your page to be valid HTML 4.01, you can use the following instead:

    <meta http-equiv="content-type" content="text/html; charset=UTF-8">
<meta http-equiv="content-type" content="text/html; charset=ISO-8859-1">
  3. Social Media

    4. Meta tagi, takie jak og:title czy twitter:card, służą do przekazywania informacji o stronie serwisom społecznościowym. Nie są one widoczne dla użytkownika odwiedzającego stronę, ale są odczytywane przez Facebooka, Twittera, LinkedIna i inne platformy.

    Dzięki nim, po wklejeniu linku do strony w poście, automatycznie pojawia się tytuł, opis i miniatura — co przyciąga uwagę i zachęca do kliknięcia.

    Przykłady

            Open Graph (Facebook, LinkedIn itp.)
<meta property="og:title" content="Tytuł treści">
<meta property="og:description" content="Opis strony, który pojawi się przy udostępnianiu linku.">
<meta property="og:image" content="https://example.com/obrazek.jpg">
<meta property="og:url" content="https://example.com/podstrona">
<meta property="og:type" content="website">
<meta property="og:site_name" content="Nazwa strony">

        Twitter
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="Tytuł treści na Twitterze">
<meta name="twitter:description" content="Krótki opis widoczny pod linkiem.">
<meta name="twitter:image" content="https://example.com/twitter-image.jpg">
    

    Na urządzeniach mobilnych (szczególnie z systemem iOS i przeglądarką Safari), numer telefonu wpisany jako zwykły tekst, np.:
<p>Skontaktuj się z nami: 123 456 789</p>
automatycznie zostaje rozpoznany jako numer telefonu i zamieniony na klikalny link typu tel:, czyli np.:
<a href="tel:123456789">123 456 789</a>
Dzięki temu użytkownik może w jedno kliknięcie zadzwonić.

    Czasami algorytmy mylą się i:

    • potraktują ISBN, numer zamówienia lub numer referencyjny jako numer telefonu,
    • co prowadzi do niepotrzebnych linków,
    • a to może wyglądać nieprofesjonalnie lub wprowadzać w błąd.

    Aby uniemożliwić przeglądarce rozpoznawanie numerów jako tel:, dodajesz w <head>:

    <meta name="format-detection" content="telephone=no">
  4. Automatic Refresh

    5. To refresh the page every five seconds, add this meta element in the head element:

    <meta http-equiv="refresh" content="5">

    CAUTION! While this is a valid command, it is recommended that you do not use it because of its negative effects on user experience. Refreshing the page too often can cause it to become unresponsive, and often scrolls to the top of the page. If some information on the page needs to be updated continuously, there are much better ways to do that by only refreshing a portion of a page.

  5. Automatic redirect

    6. Sometimes your webpage needs a automatic redirect. For example, to redirect to example.com after 5 seconds: 

    <meta http-equiv="refresh" content="5;url=https://www.example.com/" />

    This is line will send you to the designated website (in this case example.com) after 5 seconds. If you need to change the time delay before a redirect, simply changing the number right before your ;url= will alter the time delay.

  6. Web App

    7. You can set up your web app or website to have an application shortcut icon added to a device's homescreen, and have the app launch in full-screen "app mode" using Chrome for Android's "Add to homescreen" menu item. Below meta tag(s) will open web app in full-screen mode (without address bar).

    Android Chrome
    <meta name="mobile-web-app-capable" content="yes">

    IOS
    <meta name="apple-mobile-web-app-capable" content="yes">

    You can also set color for status bar and address bar in meta tag.

    Android Chrome
    <meta name="theme-color" content="black">

    IOS
    <meta name="apple-mobile-web-app-status-bar-style" content="black">
