Linking Resources

  1. Atrybuty znacznika <link>
    • charset - kodowanie znaków w dołączonym dokumencie
    • crossorigin - sposób obsługi żądań cross-origin
    • href - lokalizacja zasobu
    • hreflang - język zasobu
    • media - na jakim urządzeniu styl ma być stosowany
    • rel - relacja między dokumentem a zasobem (wymagane)
    • rev - odwrotna relacja między zasobem a dokumentem
    • sizes - rozmiar zasobu (dla rel="icon")
    • target - gdzie ma być załadowany zasób
    • type - typ MIME zasobu
    • integrity - SHA hash do weryfikacji zasobu
  2. JavaScript
    3. Synchronous (standardowo)
    <script src="path/to.js"></script>

    Skrypt ładuje się i wykonuje od razu, zanim przeglądarka przejdzie dalej z parsowaniem HTML.
    Najczęściej umieszczany tuż przed zamykającym </body>, aby nie blokować ładowania strony.

    Asynchronous
    <script src="path/to.js" async></script>

    Ładuje się równolegle i wykonuje się natychmiast po pobraniu. Może przerwać parsowanie HTML.

    Deferred
    <script src="path/to.js" defer></script>

    Skrypt ładuje się równolegle z HTML, ale czeka z wykonaniem aż cały HTML zostanie sparsowany. Gwarantuje kolejność wykonania.

    <noscript>
    <noscript>JavaScript disabled</noscript>

    Pokazuje treść, jeśli JavaScript jest wyłączony lub nieobsługiwany.

  3. External CSS Stylesheet
    4. <link rel="stylesheet" href="path/to.css">

    Najlepiej umieszczać w <head>, aby styl ładował się na początku. Jeśli natomiast zrobisz:

    <head>
    <link rel="stylesheet" type="text/css" href="general.css">
    <link rel="stylesheet" type="text/css" href="specific.css">
</head>

    W tym wypadku pamiętaj że jeśli użyjesz wielu stylów, to ten drugi (specific.css) nadpisze style w general.css (nadpisze tylko wtedy kiedy oba elementy będą się odwoływać do tego samego znacznika, id lub klasy)

  4. Alternatywa: @import w <style>
    5. <style>
  @import("path/to.css");
</style>
  5. Favicon (ikona strony)
    6. <link rel="icon" type="image/png" href="/favicon.png">
<link rel="shortcut icon" type="image/x-icon" href="/favicon.ico">

    Favicon może być automatycznie pobrany z /favicon.ico. Zmiany mogą nie być od razu widoczne przez cache.

  6. Alternate CSS
    7. <link rel="alternate stylesheet" href="path/to/style.css" title="yourTitle">

    Niektóre przeglądarki pozwalają użytkownikowi wybrać styl przez ustawienia (np. Firefox: Widok > Styl strony).

  7. Resource Hints
    8. Preconnect
    <link rel="preconnect" href="URL">

    Wykonuje DNS + TCP + opcjonalnie TLS. Eksperymentalne.

    DNS-Prefetch
    <link rel="dns-prefetch" href="URL">

    Wstępnie rozwiązuje nazwę DNS.

    Prefetch
    <link rel="prefetch" href="URL">

    Pobiera zasób wcześniej do cache.

    Prerender
    <link rel="prerender" href="URL">

    Ładuje i renderuje stronę w tle. Eksperymentalne.

  8. Media Attribute
    9. <link rel="stylesheet" href="print.css" media="print">

    Styl ładowany tylko dla wydruku lub konkretnego typu urządzenia.

  9. Prev & Next
    10. <link rel="prev" href="http://example.com/previous-page">
<link rel="next" href="http://example.com/next-page">

    Używane w artykułach lub seriach do nawigacji między stronami.

  10. Web Feed (RSS / Atom)
    11. <link rel="alternate" type="application/atom+xml" href="http://example.com/feed.xml" />
<link rel="alternate" type="application/rss+xml" href="http://example.com/feed.xml" />

    Umożliwia wykrycie kanałów RSS/Atom przez przeglądarki lub aplikacje czytające.
