ARIA

  1. Co to jest ARIA?

    2. ARIA (Accessible Rich Internet Applications) to zbiór atrybutów HTML, które dodaje się do elementów strony, aby uczynić je bardziej dostępnymi dla osób z niepełnosprawnościami, zwłaszcza korzystających z technologii asystujących, takich jak czytniki ekranu. ARIA nie zmienia wyglądu ani funkcjonalności elementów, ale dostarcza dodatkowych informacji o roli, stanie i właściwościach elementów, co pozwala czytnikom ekranu precyzyjniej opisywać zawartość strony.

  2. Landmark roles (obszary i nawigacja)
    • role="banner"

      • ➤ Zastosowanie: Główna sekcja nagłówka strony (np. logo + menu górne). Używana tylko raz.
      ➤ Dlaczego: Pomaga technologiom wspomagającym zidentyfikować nagłówek strony.

      <div role="banner">
    <h1>My Site</h1>
    <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
        <li><a href="/contact">Contact</a></li>
    </ul>
</div>
    • role="navigation"

      • ➤ Zastosowanie: Otacza główne menu lub boczne panele nawigacyjne.
      ➤ Dlaczego: Czytniki ekranu mogą łatwo przeskoczyć do nawigacji.

      <div role="navigation">
    <h1>My Site</h1>
    <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
        <li><a href="/contact">Contact</a></li>
    </ul>
</div>
    • role="main"

      • ➤ Zastosowanie: Sekcja z główną zawartością strony (np. artykuł, produkt).
      ➤ Dlaczego: Ułatwia szybkie przejście do głównej treści. 

      <!-- header & nav here -->
<div role="main">
    <p>Lorem ipsum...</p>
</div>
<!-- footer here -->
    • role="complementary"

      • ➤ Zastosowanie: Sidebar lub inne uzupełniające informacje (np. reklama, polecane artykuły).
      ➤ Dlaczego: Informuje, że treść jest poboczna wobec głównej.

      <div role="complementary">
    <h2>More Articles</h2>
    <ul>
        <!-- etc -->
    </ul>
</div>
    • role="contentinfo"

      • ➤ Zastosowanie: Stopka strony zawierająca dane kontaktowe, prawa autorskie itp.
      ➤ Dlaczego: Określa, że to końcowa część dokumentu.

      <p role="contentinfo">
    Author: Albert Einstein<br>
    Published: August 15, 1940
</p>
    • role="search"

      • ➤ Zastosowanie: Sekcja zawierająca formularz wyszukiwania.
      ➤ Dlaczego: Pozwala użytkownikom z czytnikami szybko odnaleźć wyszukiwarkę.

      <div role="search">
    <input role="searchbox" type="text">
    <button role="button">Search</button>
</div>
  3. Widget roles (elementy interfejsu)
    • role="button"

      • ➤ Zastosowanie: Element działający jak przycisk, np. <div> klikany.
      ➤ Dlaczego: Zapewnia, że czytnik traktuje element jak przycisk. 

          <button role="button">Add</button>
    • role="link"

      • ➤ Zastosowanie: Nienatywne linki np. onclick na <span>.
      ➤ Dlaczego: Umożliwia dostępność niestandardowych linków.

    • role="checkbox" / role="radio"

      • ➤ Zastosowanie: Własne komponenty formularza zamiast <input>.
      ➤ Dlaczego: Informuje o typie wyboru i stanie zaznaczenia.

      <p>
    <input type="checkbox" role="checkbox" aria-checked="false">
    <I agree to the terms
</p>
    • role="combobox"

      • ➤ Zastosowanie: Pole tekstowe z listą sugerowanych opcji.
      ➤ Dlaczego: Ułatwia dostępność dla wyszukiwarek z autouzupełnianiem.

      <input type="text" role="combobox" aria-expanded="false">
    • role="slider"

      • ➤ Zastosowanie: Interaktywny suwak wartości np. głośność.
      ➤ Dlaczego: Czytnik rozpoznaje zmianę wartości.

      <div
    role="slider"
    aria-valuemax="100"
    aria-valuemin="0"
    aria-valuenow="25">
    <div class="sliderhandle"></div>
</div>
    • role="tablist" / tab / tabpanel"

      • ➤ Zastosowanie: Niestandardowe zakładki (np. z użyciem div).
      ➤ Dlaczego: Pozwala poruszać się między zakładkami za pomocą klawiatury.

      <ul role="tablist">
    <li role="tab">Introduction</li>
    <li role="tab">Chapter 1</li>
    <li role="tab">Chapter 2</li>
</ul>
    • role="tooltip"

      • ➤ Zastosowanie: Dymek wyświetlany po najechaniu myszką.
      ➤ Dlaczego: Komunikuje pomocnicze informacje.

      <span aria-describedby="slopedesc">Slope</span>
<div role="tooltip" id="slopedesc">y=mx+b</div>
    • role="dialog"

      • ➤ Zastosowanie: Okna modalne, np. popup z formularzem.
      ➤ Dlaczego: Informuje o interaktywnym, skupionym kontekście.

      <div role="dialog">
    <p>Are you sure?</p>
    <button role="button">Yes</button>
    <button role="button">No</button>
</div>
  4. Structural roles (struktura)
    • role="article"

      • ➤ Zastosowanie: Używane tylko jeśli nie możesz użyć <article>.
      ➤ Dlaczego: Identyfikuje sekcję jako niezależny wpis (np. blog).

      <div role="article">
    <h1>My first article</h1>
    <p>Lorem ipsum...</p>
</div>
    • role="region"

      • ➤ Zastosowanie: Ważne obszary (np. pole wyników wyszukiwania).
      ➤ Dlaczego: Czytniki mogą pozwolić przeskoczyć do tej sekcji.

      <div role="region">
 Home team: 4<br>
 Away team: 2
</div>
    • role="presentation"

      • ➤ Zastosowanie: Gdy element ma tylko funkcję wizualną (np. tabela do layoutu).
      ➤ Dlaczego: Ignoruje semantykę - np. <table> nie będzie traktowane jak tabela.

      <div style="float:left;">Some content on the left.</div>
<div style="float:right;">Some content on the right</div>
<div role="presentation" style="clear:both;"></div> <!-- Only used to clear floats -->
  5. Live region roles (powiadomienia)
    • role="alert" + aria-live="assertive"

      • ➤ Zastosowanie: Pilne komunikaty np. „Uwaga! Sesja wygasa”.
      ➤ Dlaczego: Czytniki odczytują treść natychmiast po zmianie.

      <div role="alert" aria-live="assertive">Your session will expire in 60 seconds.</div>

      Note that I've included both role="alert" and aria-live="assertive" at the same time. These are
synonymous attributes, but some screen readers only support one or the other. By using both
simultaneously we therefore maximize the chances that the live region will function as expected.
    • role="status" + aria-live="polite"

      • ➤ Zastosowanie: Komunikaty niekrytyczne, np. „Zapisano zmiany”.
      ➤ Dlaczego: Czytnik odczyta treść bez przerywania użytkownikowi.

      <div role="status">Online</div>
    • role="log"

      • ➤ Zastosowanie: Dziennik zdarzeń lub historia wiadomości.
      ➤ Dlaczego: Nowe wpisy są automatycznie odczytywane użytkownikowi.

      <ul role="log">
    <li>User 1 logged in.</li>
    <li>User 2 logged in.</li>
    <li>User 1 logged out.</li>
</ul>
