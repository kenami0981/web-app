HTML5 Cache

  1. Cache w HTML5 (Application Cache)
    • Podstawowy przykład

      • W pliku HTML należy dodać atrybut manifest do tagu <html>:

      <!DOCTYPE html>
<html manifest="index.appcache">
  <body>
    <p>Content</p>
  </body>
</html>

      Następnie tworzysz plik index.appcache w tym samym folderze z treścią:

      CACHE MANIFEST
index.html

      Po wczytaniu strony przeglądarka zapisuje pliki wymienione w manifest w cache i można ją przeglądać offline.

    • Ważne informacje

      • Application Cache zostało przestarzałe (deprecated) i nie jest już zalecane do użycia w nowych projektach.

      Nowoczesnym podejściem do cache'owania offline są Service Workers i Cache API.

      Application Cache było wygodne, ale miało wiele wad, m.in. trudną kontrolę nad aktualizacjami i problemy z bezpieczeństwem.

    • Uwagi

      • Oba pliki (HTML i .appcache) muszą znajdować się w tym samym folderze.

      Po zmianie zawartości appcache konieczne jest odświeżenie cache przez przeglądarkę.

      Strona musi być serwowana z odpowiednim nagłówkiem MIME: text/cache-manifest.
