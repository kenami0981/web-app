Output Element

Element <output> w HTML służy do wyświetlania wyników obliczeń lub operacji (np. sumy dwóch liczb, wyników formularza itp.). Jest przydatny, gdy chcesz pokazać dynamiczne wyniki na stronie bez potrzeby odświeżania jej. Przyład:

<form id="form1" id="form1" oninput="out1.value = parseInt(in1.value,10) + parseInt(in2.value,10)">
    <fieldset>
        <legend>Dodawanie</legend>
        <input name="in1" id="in1" type="number" value="0"><br>
        <input name="in2" id="in2" type="number" value="0">
    </fieldset>
    <output form="form1" name="out1" for="in1 in2">0</output>
</form>

Efekt:

Dodawanie
0

Atrybut Opis
name Nazwa outputu (jak w <input>). Pozwala odnieść się do elementu np. w JavaScript lub przy wysyłce formularza.
for Lista ID elementów formularza (np. input, select), których wartości będą wpływać na ten output.
form ID formularza, z którym ma być powiązany (jeśli nie znajduje się bezpośrednio w tym formularzu).

Dodatek

Void Elements

Void elements to specjalna grupa znaczników HTML, które nie zawierają żadnej treści i dlatego nie wymagają znacznika zamykającego. Używa się ich wyłącznie z otwierającym tagiem, np. <br>.

Cechy void elements:

  1. Nie zawierają treści ani elementów wewnętrznych.
  2. Nie wymagają znacznika zamykającego.
  3. W HTML5 nie trzeba dodawać ukośnika (/) - ale jest to tolerowane.

Przykłady void elements:

Znacznik Opis
<br> Łamanie linii
<img> Obrazek
<input> Pole formularza
<hr> Pozioma linia
<meta> Metadane dokumentu
<link> Dołączanie CSS lub innych zasobów
<area> Klikalny obszar w mapie obrazka
<base> Bazowy URL dla linków
<source> Źródło dla video/audio
<param> Parametry dla osadzanych treści

Poprawne użycie:

<img src="logo.png" alt="Logo">
<br>
<hr>
<input type="text" placeholder="Wpisz swoje imię">

Niepoprawne użycie:

</br>       ← Błąd
<img></img> ← Błąd

Dodatkowe uwagi:

  1. W XHTML używa się <br />, <img /> - to wymóg składni XML.
  2. W HTML5 zaleca się pisać po prostu <br> bez zamykania ukośnikiem.
