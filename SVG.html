SVG

  1. Definicja SVG

    2. SVG (Scalable Vector Graphics) to skalowalne grafiki wektorowe oparte na XML, używane do tworzenia grafiki w przeglądarce.

    Element <svg> jest kontenerem dla grafiki SVG - może zawierać kształty, tekst, linie, ścieżki i inne elementy graficzne.

  2. Inline SVG

    3. SVG może być zapisany bezpośrednio w dokumencie HTML. Przykład:

  3. Osadzanie zewnętrznego pliku SVG

    4. <img> lub <object> mogą ładować zewnętrzne pliki SVG.

    <img src="attention.svg"> - nie umożliwia stylowania ani manipulacji JS.

    <object type="image/svg+xml" data="attention.svg"> - umożliwia stylowanie i interakcje.

  4. Dlaczego warto korzystać z SVG (zamiast <img>)?
    • Skalowalność - nie traci jakości w żadnej rozdzielczości (Retina, druk, zoom).
    • Stylowanie CSS-em - można zmieniać kolor, rozmiar, dodać hover itp.
    • Mały rozmiar - idealne dla ikon i UI (czysty kod XML zamiast dużych bitmap).
    • Dostępność - SVG można oznaczać ARIA/alt, co poprawia dostępność.
    • Animacje - wspiera animacje CSS oraz SMIL/JS.
    • Manipulacja JS - można dynamicznie zmieniać SVG w locie (np. kolor, położenie).

    W skrócie: SVG = lepsze ikony, grafiki, interaktywne elementy UI. <img> to tylko obraz.

  5. Potencjał SVG - przykładowa grafika
    6. <svg viewBox="0 0 100 100" class="sparkle">
  <polygon points="50,10 60,40 90,40 65,60 75,90 50,70 25,90 35,60 10,40 40,40" />
</svg>
    .sparkle {
    fill: gold;
    stroke: black;
    stroke-width: 2;
    width: 50px;
    height: 50px;
}
.sparkle:hover {
    fill: orange;
}

    Ten element zmienia kolor po najechaniu - dzięki temu idealnie nadaje się np. do interfejsu użytkownika.

  6. Podsumowanie
    • SVG daje więcej możliwości niż <img>.
    • Warto go znać do projektowania UI, ikon, animacji i dynamicznych grafik.
    • SVG może być interaktywny, responsywny i dostępny.
